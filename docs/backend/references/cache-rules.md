# CACHE - 缓存规则

## CACHE-001 Key 命名

**等级**: 🟡 警告

```
edms:{module}:{entity}:{id}
edms:upms:user:1001
edms:upms:dict:type:status
```

- 全小写，冒号分隔
- 禁止无模块前缀的裸 key（如 `user:1001`）

## CACHE-002 适用场景

**等级**: 🟡 警告

| 适合缓存 | 不适合缓存 |
|----------|------------|
| 字典、配置、菜单树 | 强一致余额、库存 |
| 读多写少的用户摘要 | 刚写入立即读的关键路径（或短 TTL + 失效） |
| 验证码、Token 黑名单 | 大对象列表全量缓存 |

## CACHE-003 一致性

**等级**: 🔴 严重

- 写操作（insert/update/delete）必须**删除或更新**对应缓存，禁止只写 DB 不失效缓存
- 缓存穿透：对不存在 key 可缓存空值（短 TTL）或布隆过滤（按需）

## CACHE-004 TTL

**等级**: 🔵 建议

| 数据类型 | 建议 TTL |
|----------|----------|
| 字典/配置 | 30min ~ 24h |
| 用户权限摘要 | 5 ~ 15min |
| 验证码 | 5min 以内 |

## CACHE-005 工具与配置

**等级**: 🟡 警告

- 使用 `edms-common-data` 提供的 Redis 配置
- 禁止在业务代码中硬编码 Redis 地址
- 序列化方式与项目统一配置保持一致
